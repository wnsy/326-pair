Midpoint

  1. Integer overflow is when we have a number that fits in 32 bits or 64 bits but if we add to it (or multiply it by something, or do another operation), the answer would not fit the original 32 or 64 bits.

  2. How to detect it: In lower-flevel language such as C, the processor does "its best" taking the true result, will then get rid of any bits that don't fit.

  3. How to prevent it: use larger integer types in C:
    long long int
    or in Java: 
    BigInteger
